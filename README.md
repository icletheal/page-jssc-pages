# page-jssc-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面彼此独立，不依赖特定的域名或网站环境，适合作为静态页面集合进行管理和访问。

## 项目简介

`page-jssc-pages` 是一个轻量级的页面存档仓库。主要目的是将各类独立 HTML 页面集中存放，并通过 GitHub Pages 或其他静态托管服务进行发布。每个页面均为自包含的 HTML 文件，不涉及后端服务或动态内容。

## 目录说明

```
.
├── index.html          # 仓库首页 / 页面导航
├── pages/              # 页面归档目录
│   ├── example1.html   # 示例页面
│   ├── example2.html
│   └── ...
├── assets/             # 公共资源（可选）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `index.html`：可选的主页，可列出所有归档页面的链接。
- `pages/`：存放所有独立 HTML 页面的目录。每个文件对应一个页面。
- `assets/`：存放页面共用的样式、脚本或图片资源（如需要）。

## 页面归档说明

- 每个页面均为完整的 HTML 文档，包含独立的 `<head>` 和 `<body>`。
- 页面之间没有强依赖关系，可单独打开访问。
- 归档内容可能包括但不限于：信息展示页、工具页、实验性页面等。
- 不保证所有页面均经过全面测试或长期维护。

## 使用方式

1. 克隆或下载本仓库。
2. 在本地直接用浏览器打开 `pages/` 下的 HTML 文件即可查看。
3. 若需通过 GitHub Pages 发布，请确保仓库设置中已启用 Pages 功能，并选择合适的分支和目录。
